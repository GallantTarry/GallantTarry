少侠，没问题。Obsidian 是一件非常趁手的知识管理兵器，它的语法大部分遵循标准 Markdown，同时也加入了一些极其强大的独占扩展（比如双链和标注）。

这里为你系统梳理一下日常做笔记和沉淀技术文档最常用的排版语法：

### 1. 标题 (Headings)

在文字前加上 `#` 和一个空格，就可以创建标题。`#` 的数量代表标题的层级，最多支持六级。

Markdown

```
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

### 2. 文本格式 (Text Formatting)

Obsidian 支持标准的文本修饰，其中**高亮**是其非常实用的特色功能。

|**效果**|**语法**|**示例**|
|---|---|---|
|**加粗**|`**文字**`|`**核心逻辑**`|
|_斜体_|`*文字*`|`*注意事项*`|
|~~删除线~~|`~~文字~~`|`~~废弃方案~~`|
|==高亮==|`==文字==`|`==重点标红==`|

### 3. 列表 (Lists)

列表可以互相嵌套，只需要在换行后按 `Tab` 键缩进即可。

**无序列表：**

使用 `-`、`*` 或 `+` 加空格。

Markdown

```
- 第一项
- 第二项
  - 子项 (按 Tab 缩进)
```

**有序列表：**

使用数字加英文句号和空格。

Markdown

```
1. 第一步：初始化环境
2. 第二步：配置依赖
```

**任务列表 (Task Lists)：**

非常适合用来做待办事项追踪。

Markdown

```
- [ ] 未完成的任务
- [x] 已完成的任务
```

### 4. 代码块 (Code Blocks)

作为日常沉淀代码片段的工具，代码块必不可少。

**行内代码：**

使用反引号包裹，适合标记类名、方法名或短命令。
键盘左上方波浪。

语法：`` `String` ``

效果：`String`

**多行代码块：**

使用三个反引号包围，并在第一行末尾指定编程语言（如 java, sql, bash 等）以实现语法高亮。

Markdown

````
```java
@RestController
@RequestMapping("/api")
public class TestController {
    // 你的代码
}
````

````

### 5. 引用与标注 (Quotes & Callouts)
**普通引用：**
在文字前加 `>`。
```markdown
> 放弃对“术”的沉迷，追求底层的“道”。
````

**Obsidian 专属标注 (Callouts)：**

这是 Obsidian 中极其美观的功能，可以在引用块中加上特定的标记来改变样式。支持的类型有很多，比如 `[!note]`, `[!warning]`, `[!info]`, `[!danger]` 等。

Markdown

```
> [!warning] 注意
> 这里的数据库连接需要隔离，否则会报错。

> [!tip] 提示
> 可以使用快捷键 Cmd/Ctrl + E 快速切换编辑/阅读模式。
```

### 6. 链接 (Links)

**双向链接 (Obsidian 的灵魂)：**

使用双方括号，可以快速链接到你的其他笔记页面。

Markdown

```
[[另一篇笔记的名称]]
[[笔记名称#某个特定的标题]]
```

**外部链接：**

链接到网上的网页。

Markdown

```
[链接显示的文字](https://www.example.com)
```

### 7. 分隔线 (Horizontal Rules)

连续输入三个或以上的 `-` 或 `*`，并独占一行，可以生成一条分割线，用来划分文档区块。

Markdown

```
---
```

**快速上手建议：** 少侠可以直接将上面的语法复制到 Obsidian 的新建笔记里，然后按下 `Ctrl + E`（或点击右上角的一本书/笔的图标）在“阅读视图”和“编辑视图”之间切换，立刻就能直观地看到所有排版效果。
